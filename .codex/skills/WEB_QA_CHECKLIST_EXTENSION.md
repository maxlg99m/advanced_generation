# WEB_QA_CHECKLIST_EXTENSION

## Назначение

Расширяет базовые правила для web-приложений и browser-based интерфейсов.

## Когда подключать

- ТЗ или запрос указывает на web-приложение.
- Дизайн описывает desktop web UI.
- Требуется browser-based продукт.

## Platform

Для web использовать только:

```text
Web (Desktop)
```

Значение выводится без исправлений, сокращений и альтернативных вариантов.

## Обязательное покрытие

- URL-навигация.
- Refresh.
- Back/forward.
- Открытие в новой вкладке.
- Обход ограничений через URL/query params.
- Browser storage: `localStorage`, `sessionStorage`, `cookies`, если применимо.
- Web cache/service worker, если явно используется.
- Кроссбраузерность только там, где это релевантно.

## Кроссбраузерные проверки

- Для каждого браузера создаётся отдельный чек-лист.
- Один браузер = один чек-лист.
- Объединение нескольких браузеров в одном чек-листе запрещено.
- Название браузера обязательно указывается в `Title`.

## Ограничения

Этот skill не заменяет `MASTER_QA_SYSTEM_PROMPT` и не переопределяет CSV-контракт.
