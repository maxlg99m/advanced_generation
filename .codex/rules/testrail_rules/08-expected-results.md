# Expected Result rules

`Expected Result`:

- содержит конкретное ожидаемое поведение;
- содержит конкретное UI-состояние, если проверка затрагивает интерфейс;
- содержит трассировку на требование, раздел ТЗ или дизайн-источник;
- содержит конкретный текст ошибки, если он указан во входных материалах;
- содержит конкретный HTTP-код при серверных ошибках, если он указан во входных материалах;
- не содержит выдуманных текстов ошибок, лимитов и HTTP-кодов;
- не содержит слова `корректно`, `правильно`, `успешно` без уточнения;
- не содержит размытые формулировки.
