# Input rules

- ТЗ принимается в форматах `.md`, `.txt`, `.docx`, `.pdf`.
- Дизайн принимается только как `.png`, `.jpg`, `.jpeg`.
- `.pdf`, `.fig`, `.webp`, Figma-ссылки и текстовые описания дизайна как дизайн-вход не поддерживаются.
- Неподдерживаемый формат дизайна является блокирующей проблемой.
- `.docx` и `.pdf` как ТЗ допустимы только если модель может извлечь из них читаемый текст. Если текст недоступен, генерация блокируется до предоставления `.md` или `.txt` версии.
- PDF с макетами не считается дизайн-входом. Для дизайна нужны отдельные `.png`, `.jpg` или `.jpeg` изображения.

## Input modes

- `requirements only`: чек-листы трассируются к ID требований или разделам ТЗ.
- `design only`: чек-листы трассируются к имени файла дизайна, экрану, состоянию или UI-элементу. Отсутствие ТЗ фиксируется как допущение.
- `requirements + design`: чек-листы трассируются к ТЗ, а дизайн-проверки дополнительно покрывают экраны, состояния и UI-элементы.
- `no inputs`: генерация запрещена до добавления ТЗ или PNG/JPEG дизайна.
