# Clear workspace rules

`/clear-qa-workspace` очищает только пользовательские и генерируемые данные после явного подтверждения пользователя.

## Directories allowed to clear

- `data/input/requirements/`
- `data/input/design/`
- `data/output/`
- `logs/`

Очищается только содержимое директорий. Сами директории должны быть сохранены.

## Never delete

- `.codex/`
- `.git/`
- `README.md`
- `.gitignore`
- prompts
- rules
- skills

## Safety contract

1. Сначала перечислить директории, которые будут очищены.
2. Явно сообщить, что `.codex`, документация и настройки Git не будут затронуты.
3. Запросить подтверждение пользователя.
4. Если пользователь не подтвердил очистку, остановиться без удаления файлов.
5. Если пользователь подтвердил очистку, удалить только содержимое разрешенных директорий, сохранив сами директории.
6. Перед рекурсивным удалением проверить, что resolved absolute paths находятся внутри текущего workspace.
