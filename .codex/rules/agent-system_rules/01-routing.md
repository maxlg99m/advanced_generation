# Routing rules

- `/init-qa-context` только загружает контекст и не запускает генерацию.
- `/generate-qa-checklists` запускает полный пайплайн только после успешного `/init-qa-context`.
- `/clear-qa-workspace` очищает рабочие пользовательские данные только после явного подтверждения.
- Если входные данные указывают на web, подключается `WEB_QA_CHECKLIST_EXTENSION`.
- Если входные данные указывают на mobile, подключается `MOBILE_QA_CHECKLIST_EXTENSION`.
- Если платформа неочевидна, её нужно определить из ТЗ, PNG/JPEG дизайна и пользовательского запроса.
